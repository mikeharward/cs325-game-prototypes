Crazy Snake

  • Originality: The snake gameplay is not very original, it is the same core game that has been around for decades. The original part is the control scheme, instead of the classic arrow keys you now are met with this seemingly awkward mouse guiding system. You have to try to lead the snake instead of directing controlling it, which is a fun twist on the classic snake game.
  • Technical Merit: The most interesting technical thing about my game is how the snake determines which way to move based on where the mouse is. I had to calculate a vector from the snake head to the mouse cursor and figure out which direction that vector points, then map it to the closest cardinal direction to determine the snake's movement.
  • Prototype Postmortem: The example snake game that I based this on used sub-classes for the snake and food, which made a lot of sense from an object oriented perspective but the syntax was fairly confusing as it was difficult to modify parameters from outside the class, e.g. in the update function. I had to be creative about how I designed things, keeping this in mind. I had been making global variables as class variables of the scene, but those are inaccessible inside of the subclasses, so I had to rethink some things.

  • High Concept: This snake is out of control! Can you keep it alive?
  • Theme: I used the III challenege, since there are only ever two things on the screen, the snake and the target food.
  • Mandated Variety: Input: Indirect, Mouse
    Randomness: A lot of Randomness
    Play Styles: Director
    Genres: Overhead
  • Prototype Goal: The main mechanic is the directing of the snake with the mouse. It also incorporates the random move speed gain/loss whenever collecting an apple, but the main mechanic being tested is the leading the snake with the mouse.
  • Player Experience Goals: I want there to be tension and for the player to feel a little overwhelmed, like they know what they want but the dexerity required to excute is difficult. Instead of in normal snake where you are trying to think ahead and not trap yourself, this game is focused more on trying to keep control and get the snake to go where you want.
  • Gameplay: The only control is through the mouse, which the player uses to lead the snake. Running into the walls or the snake's body will end the game. The player attempts to collect apples and grow longer, which also slowly makes the snake speed up over time, although the speed gain is random and it can lose speed at some points.
  • Strategies: The most effective strategy is trying to line up the snake on the apple from a distance, so the player is able to easily walk up to it and collect it without having to do a last-second turn. There may be some strategy in how the snake moves, e.g. always giving yourself an escape path and coiling to take up less space.
  • Story/Setting/Premise: Since it is a simple snake game, there isn't any story or premise on display here. In theory a full game could include multiple levels with varied terrain, and have the player follow a story about the snake or try to complete a longer objective, but at the current state there is no real story elements.
  • Target Audience: People who enjoy snake but want a twist on the classic game.
  • Play Time: Under a minute, but the game can go on for quite awhile.
